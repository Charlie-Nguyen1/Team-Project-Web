## e) Cloud Deployment Instructions

### Option 1: Render (Recommended - Easiest)

Render provides automatic deployments from GitHub with free tier option.

#### Step 1: Go to Render Dashboard
1. Visit **https://render.com**
2. Sign up or log in with GitHub account
3. Click **New +** ‚Üí **Web Service**

#### Step 2: Connect GitHub Repository
- Select **Team-Project-Web** repository
- Choose branch: `main`
- Render will auto-detect your `render.yaml` configuration

#### Step 3: Verify Configuration
Render will automatically configure based on `render.yaml`:
- **Build Command**: `cd Dental_App/mern_skeleton && npm run install-all && npm run build`
- **Start Command**: `cd Dental_App/mern_skeleton && npm start`
- **Environment Variables**: Pre-configured in render.yaml

#### Step 4: Deploy
- Click **Create Web Service**
- Render will automatically:
  - Clone your repository
  - Install dependencies
  - Build your frontend
  - Start the server
  - Deploy to production

**Expected Deployment Time**: 5-10 minutes

#### Step 5: Get Your Live URL
Once deployment completes, your app will be live at:
```
https://dental-app-xxxxx.onrender.com
```

cd Dental_App/mern_skeleton

railway up

#### Vercel (Frontend only)
```bash

npm install -g vercel

# Deploy frontend
cd Dental_App/mern_skeleton/client
vercel
```
Backend would need separate deployment to Render or Railway.

### View Logs in Render Dashboard
1. Go to your service dashboard

### Common Issues & Solutions

| Issue | Solution |
|-------|----------|
| Build fails | Check build command in render.yaml |
| Port mismatch | Ensure PORT env var is set to 5000 |
| DB connection fails | Verify MONGODB_URI in environment |
| Slow deploy | Free tier has limited resources; upgrade to Starter |
